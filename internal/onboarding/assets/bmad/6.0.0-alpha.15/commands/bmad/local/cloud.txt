## 🔐 CRITICAL SECURITY PROTOCOLS

### Credential Censorship (ABSOLUTE ENFORCEMENT)

**NEVER display in output:**

**AWS Secrets:**
- ❌ AWS Access Keys (AKIA*, ASIA*)
- ❌ AWS Secret Keys
- ❌ Session Tokens
- ❌ Account IDs (12-digit numbers)

**OCI Secrets:**
- ❌ OCIDs (ocid1.*.oc1..*) - Replace with aliases
- ❌ Tenancy OCIDs
- ❌ User OCIDs
- ❌ Compartment OCIDs
- ❌ Private Keys (OCI API keys)
- ❌ Auth Tokens
- ❌ OCIR credentials

**General Secrets:**
- ❌ API Keys
- ❌ Passwords
- ❌ Private Keys
- ❌ Certificates (private portions)
- ❌ Database connection strings with credentials

**Censorship Rules:**

**AWS Censorship:**
1. **Account IDs**: Always replace with aliases from account mapping
   - Example: `123456789012` → `[ACCOUNT:production]`
   - Example: `987654321098` → `[ACCOUNT:development]`

2. **Credentials**: Never print, always reference by profile
   - Example: `AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE` → `[CREDENTIALS:aws-profile-prod]`
   - Example: Token in output → `[SESSION_TOKEN:REDACTED]`

3. **CLI Output Sanitization**: Before showing ANY AWS CLI output, scan and censor:
   - Account IDs → Replace with [ACCOUNT:alias]
   - Access Keys → [ACCESS_KEY:REDACTED]
   - Secret Keys → [SECRET_KEY:REDACTED]
   - ARNs with account IDs → arn:aws:service:region:[ACCOUNT:alias]:resource

4. **Cost Reports**: Replace account IDs with aliases in all cost analysis

**OCI Censorship:**
1. **OCIDs**: Always replace with aliases from OCI mapping
   - Tenancy: `ocid1.tenancy.oc1..xxx` → `[TENANCY:alias]`
   - Compartment: `ocid1.compartment.oc1..xxx` → `[COMPARTMENT:alias]`
   - Instance: `ocid1.instance.oc1..xxx` → `[INSTANCE:name-or-alias]`
   - Other resources: `ocid1.<type>.oc1..xxx` → `[<TYPE>:alias]`

2. **OCI Credentials**: Never print
   - Auth tokens → `[OCI_AUTH_TOKEN:REDACTED]`
   - API Keys → `[OCI_API_KEY:REDACTED]`
   - OCIR passwords → `[OCIR_CREDENTIALS:REDACTED]`

3. **OCI CLI Output Sanitization**: Before showing ANY OCI CLI output, scan and censor:
   - All OCIDs → Replace with [TYPE:alias]
   - Private IPs → Mask last octet if internal
   - Fingerprints → `[FINGERPRINT:REDACTED]`

## Troubleshooting

### Common AWS CLI Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `ExpiredToken` | Session token expired | Re-authenticate: `aws sso login --profile [PROFILE]` |
| `AccessDenied` | Missing IAM permissions | Check IAM policy for required actions |
| `InvalidParameterValue` | Wrong parameter format | Verify parameter syntax in AWS docs |
| `ResourceNotFoundException` | Resource doesn't exist | Verify resource ID/name and region |
| `ThrottlingException` | API rate limit hit | Wait and retry, or request limit increase |

### Common OCI CLI Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `NotAuthenticated` | Invalid/expired token | Check `~/.oci/config` and API key |
| `NotAuthorizedOrNotFound` | Missing policy or wrong OCID | Verify compartment policies |
| `ServiceError 404` | Resource not found | Verify OCID and region |
| `ServiceError 429` | Rate limit exceeded | Implement exponential backoff |
| `InvalidParameter` | Wrong parameter format | Check OCI CLI documentation |

### Connectivity Issues

1. **AWS**: Verify `~/.aws/credentials` and `~/.aws/config` exist
2. **OCI**: Verify `~/.oci/config` and private key file permissions (should be 600)
3. **Both**: Check network/proxy settings if behind firewall
