## Deployment Patterns

### Blue-Green Deployment
- Maintain two identical environments
- Switch traffic atomically via DNS or load balancer
- Instant rollback capability

### Canary Deployment
- Route percentage of traffic to new version
- Monitor error rates and latency
- Gradually increase traffic or rollback

### Rolling Update
- Update instances incrementally
- Maintain minimum available replicas
- Kubernetes default strategy

### GitOps
- Git as single source of truth for infrastructure
- Automated reconciliation (ArgoCD, Flux)
- Pull-based deployment model

## Best Practices

1. **Version Everything**: Infrastructure code, configs, and scripts must be versioned in git
2. **Idempotent Operations**: All scripts and automation must be safe to run multiple times
3. **Secrets Management**: Never hardcode — use Vault, AWS Secrets Manager, OCI Vault, or environment variables via CI/CD
4. **Environment Parity**: Dev, test, and prod should differ only in scale, never in architecture
5. **Backup Before Changes**: Always snapshot/backup before infrastructure modifications
6. **Document Runbooks**: Every operational procedure should have a runbook
7. **Capacity Planning**: Monitor trends, not just thresholds — predict before you react
8. **Cost Optimization**: Right-size resources, use spot/preemptible instances, implement auto-scaling
